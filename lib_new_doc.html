Adding a Document to infra.sarai.net/lib/

Logging In

Go to the bottom of the homepage of infra.sarai.net/lib/ and click on Login.

Enter your username and password in the following login page.

Existing Collections

All documents kept in the library portal are part of a collection. Collections are one way of organising the documents stored in the library. The other way of organising documents is by using tags. We will talk about tags later.

Before adding a document to the library portal, please identify which category it should be part of. A document can only be part of a single category. To see the existing categories, please click on the Collection Tree on the navigation panel at the right side of the screen.

The Collection Tree screen shows the existing collections and their hierarchies (which collection is part of which collection). If the new document can be part of any of the existing collections then proceed to Adding a Document. If the new document cannot be located within any of the existing collections and you want to create a new collection, then proceed to Creating a Collection.

Creating a Collection

To create a new collection, click on the Collections tab on the navigation panel. This takes you to the list of existing collections. Unlike the Collection Tree screen, this does not show the collection hierarchies. Click on Add a Collection to create a new collection.

The portal runs on a software called Omeka, which uses Dublin Core meta-data standards to collect and keep information about the documents stored in the portal. Click here to read more about Dublin Core standards.

The Add a Collection screen has a series of text boxes to enter information about the collection concerned. It is important to fill up the following boxes (help in parentheses): Title (name of the collection), Description (a single sentence description of the collection), and Creator (enter 'Sarai-CSDS'). The rest of the boxes can be kept empty.

After filling in the details for the above mentioned boxes, click on Parent Collection tab to go to the screen below and select whether the new collection should be part of any already existing one or not. Finally, choose if the collection should be Public and/or Featured, and then click on Add Collection.

Once created, each collection gets its own page. The entered details can be reviewed and edited from here. Click on the View Public Page to see how the collection appears to visitors to the portal.

Adding a Document

In Omeka (the software that powers the library portal), each document is called an item. To add a new document to the library click on the Items tab on the navigation panel, and then click on Add an Item button in the Items page.

The Add an Item page is quite similar to the Add a Collection page discussed above. Omeka follows Dublin Core metadata standards to store information about each item.

The page has a series of text boxes to enter various information about the document being added. The important text boxes to fill up are the following (help in parentheses): Title (name of the document), Subject (keywords relevant for the document, separate by comma), Description (brief description of the document, maybe excerpt from the abstract), Creator (name of the author(s) of the document, written as -- "second name, first name"), Source (bibliographic/publication details), Publisher (name of the publisher), Date (date of publication), Rights (enter "As in original"), Language (language of the document). Other text boxes can ignored.

Click on the Item Type Metadata tab to enter the item type of the document being added. Click on the drop-down menu to select from the given options. In most cases, the added document will be of the type 'Text'.

Click on the Files tab to upload and attach the document being added. The maximum file size limit is set at 30 MB per file. If uploading a larger file, please break the original file into multiple parts and upload them separately.

Click on the Tags tab to enter related keywords (tags) for the document being added. Tags are crucial for allowing organising the documents in the portal under various themes and topics. When writing on the Add Tags text box, tags with similar spelling are automatically shown. If appropriate, please select among existing tags, so as to minimise duplication, especially those due to difference in spelling (such as "Social-Media", "SocialMedia" and "Social Media").

Finally, click on the drop-down menu on the right hand side box to locate the document within a collection. As mentioned earlier, a document can only be part of a single collection. Choose if the document should be Public and/or Featured, and click on Add Item to add the document.

One added, each document gets a separate page, where the entered details can be reviewed and edited.
